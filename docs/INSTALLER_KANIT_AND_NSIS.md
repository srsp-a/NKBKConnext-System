# ทำไมหน้าติดตั้ง (NSIS) ใช้ฟอนต์ Kanit ไม่ได้

## ข้อจำกัดของเครื่องมือ

ตัวติดตั้งที่สร้างจาก **electron-builder + NSIS** เป็นหน้าต่าง **Windows แบบดั้งเดิม** (Win32 / Modern UI):

- ข้อความ ปุ่ม หัวข้อ ใช้ **ฟอนต์ระบบ** (เช่น Segoe UI) เท่านั้น
- **ไม่มีทาง** ฝังฟอนต์ Kanit หรือ Google Fonts ในข้อความของ wizard ได้
- แถบภาพซ้าย / มุม header ที่ปรับได้คือ **รูป BMP** เท่านั้น (โทนสี + โลโก้) — ไม่ใช่ HTML/CSS

ดังนั้น **UI แบบเดียวกับแอป Electron** (Kanit, มุมโค้ง, dark card) **ทำในตัวติดตั้ง NSIS แบบเต็มรูปแบบไม่ได้** โดยไม่เปลี่ยนเทคโนโลยีทั้งก้อน

## ทางเลือกถ้าต้องการ “ทันสมัย + Kanit” จริงๆ

| ทางเลือก | ข้อดี | ข้อเสีย |
|----------|--------|---------|
| **MSIX** (Microsoft Store / แพ็กเก์ App Installer) | หน้าติดตั้งของ Windows 11 ดูใหม่กว่า | ต้องลงนาม / โฟลว์ต่างจาก NSIS |
| **ติดตั้งแบบเงียบ + โฟกัส UI ในแอป** | แอปใช้ Kanit ครบ — ติดตั้งแค่ครั้งเดียว | ตัว Setup ยังเป็นของระบบ |
| **Wizard แยกด้วย Electron/HTML** | ออกแบบได้เหมือนเว็บ | ไฟล์ใหญ่ ซับซ้อน ดูแลยาก |
| **ภาพ BMP ที่มีข้อความ Kanit “ฝัง” ในภาพ** | เห็นตัวหนังสือ Kanit ในภาพ | แก้ข้อความยาก, หลายภาษาไม่สะดวก |

## ในโปรเจกต์นี้ทำอะไรได้แล้ว

- ภาพ **sidebar / header** ให้โทนใกล้แอป (เข้ม + เขียว accent)
- หน้า **Welcome** แสดงแถบภาพเต็ม
- หน้า **`public/installer-guide.html`** — เปิดในเบราว์เซอร์ได้ ใช้ **Kanit + ธีมเดียวกับแอป** สำหรับคำแนะนำก่อน/หลังติดตั้ง (ไม่ใช่ส่วนของ NSIS)

สรุป: **แอปหลัก = Kanit ได้เต็มที่** — **Setup.exe ของ NSIS = ข้อจำกัดของ Windows**
