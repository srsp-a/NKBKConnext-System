# หลักการตรวจสอบการล็อกอิน และการนำไปใช้ในโปรแกรม

## 1. หลักการตรวจสอบโดยทั่วไป (ระบบตรวจสอบจากอะไร ที่ไหน ยังไง)

ระบบล็อกอินบนเว็บส่วนใหญ่ทำงานในลักษณะเดียวกัน:

| หัวข้อ | รายละเอียด |
|--------|-------------|
| **ตรวจสอบจากอะไร** | คู่ **ชื่อผู้ใช้ (username)** และ **รหัสผ่าน (password)** ที่ผู้ใช้กรอกในฟอร์ม บางระบบมีฟิลด์เพิ่ม (เช่น เลือกฐานข้อมูล, จดจำฉันไว้) |
| **ตรวจสอบที่ไหน** | ฝั่ง **เซิร์ฟเวอร์** หลังผู้ใช้กดปุ่มส่งฟอร์ม (เช่น "เข้าสู่ระบบ" / "เข้าใช้งาน") เบราว์เซอร์จะส่งคำขอ (มักเป็น **POST**) ไปยัง **URL ของฟอร์ม** (ค่า `action` ของ `<form>`) หรือ endpoint ที่สคริปต์บนหน้ากำหนด |
| **ตรวจสอบยังไง** | เซิร์ฟเวอร์รับค่า username/password แล้วเทียบกับข้อมูลใน **ฐานข้อมูล หรือระบบสมาชิก** (เช่น LDAP, Active Directory) ถ้าตรง → สร้าง **session / cookie** แล้ว redirect ไปหน้าหลัก ถ้าไม่ตรง → แสดงข้อความผิดพลาด |

สรุปสั้นๆ:
- ** input:** username + password (และอาจมี database, remember me)
- **ที่ส่งไป:** URL ตาม `form.action` หรือที่สคริปต์กำหนด (มัก POST)
- **ฝั่งเซิร์ฟเวอร์:** ตรวจจาก DB/สมาชิก → ออก session/cookie

เพื่อให้โปรแกรมเรา “เปิดในแอป + เติม user/pass ให้อัตโนมัติ” ได้ ต้องรู้ว่าใน **หน้า HTML**:
- ฟอร์มอยู่ที่ไหน (`form` หรือ `form#xxx`)
- ช่อง user/pass ใช้ **id หรือ name** อะไร (หรืออยู่ติดกับ label ข้อความอะไร)
- ปุ่มส่งฟอร์มเป็นปุ่มไหน (id/name/value หรือข้อความบนปุ่ม)

จากนั้นในแอปเราจึง **inject สคริปต์** เพื่อ: เติมค่าช่อง user/pass (และ database ถ้ามี) แล้ว **ส่งฟอร์ม** (เช่น `form.submit()` หรือ **คลิกปุ่มส่ง**) ให้ตรงกับวิธีที่เว็บนั้นใช้

## 2. เว็บที่เกี่ยวข้องกับโปรแกรมนี้

### 2.1 oa.nkbkcoop.com/nkh (ระบบเดิมที่รองรับอยู่)

- ฟอร์มใช้ **id/name**: `TbUsername`, `TbPassword`, ปุ่ม `BtLogIn`, มี `<select>` สำหรับฐานข้อมูล
- ปุ่มส่ง: มักเป็น "เข้าใช้งาน"
- สคริปต์ใน `electron-main.js` ออกแบบมาให้รองรับฟอร์มรูปแบบนี้เป็นหลัก (รวมถึง fallback หาจาก label "ชื่อผู้ใช้", "รหัสผ่าน")

### 2.2 https://led-ck.com/ (Login)

จากข้อมูลที่มี (หน้า Login ที่ดึงได้):

- **ข้อความบนหน้า:** ชื่อผู้ใช้งาน, รหัสผ่าน, จดจำฉันไว้, ปุ่ม **เข้าสู่ระบบ**, ลิงก์สมัครสมาชิก (register)
- **หลักการตรวจสอบ:** เหมือนระบบทั่วไป — ระบบตรวจจาก **username + password** ที่กรอก แล้วส่งไปตรวจที่ **เซิร์ฟเวอร์ของ led-ck.com** (ที่ URL ที่ฟอร์มส่งไป) ตรวจจาก **ฐานข้อมูล/ระบบสมาชิกของเว็บนั้น** แล้วออก session/cookie
- **สิ่งที่โปรแกรมต้องทำเพื่อให้ “เปิดในแอป” ใช้ได้:**  
  - หาช่อง **ชื่อผู้ใช้** (อาจใช้ label "ชื่อผู้ใช้งาน" หรือ id/name ของ input)  
  - หาช่อง **รหัสผ่าน**  
  - หาปุ่ม **เข้าสู่ระบบ**  
  แล้วเติม user/pass และกดส่งฟอร์ม (หรือคลิกปุ่ม)

เนื่องจากเราไม่ได้เห็น HTML จริงของ led-ck.com (มีแต่ข้อความจาก fetch) การรองรับในแอปจึงทำแบบ **ทั่วไป**:
- หาช่อง user จาก label ที่มี "ชื่อผู้ใช้" หรือ "ชื่อผู้ใช้งาน" หรือ "username"
- หาช่อง pass จาก label "รหัสผ่าน" หรือ "password" หรือ `input[type="password"]`
- หาปุ่มส่งจากข้อความปุ่ม "เข้าใช้งาน" **หรือ "เข้าสู่ระบบ"**

ถ้าต้องการให้ตรงกับ led-ck.com แม่นยำมากขึ้น แนะนำให้เปิด DevTools ที่หน้า Login ของ led-ck.com แล้วดู:
- **Elements:** tag `<form>`, attribute `action`, และ id/name ของ input แต่ละตัวกับปุ่มส่ง
- **Network:** กด "เข้าสู่ระบบ" แล้วดู request ที่ส่งออก (URL, method, form body) ว่าไปที่ไหน ส่งค่าอะไร

## 3. การนำไปใช้ในโปรแกรม (System Status Monitor)

- **การ์ด “เชื่อมต่อกับเว็บไซต์”** ในแอป: ผู้ใช้กรอก URL, ฐานข้อมูล (ถ้ามี), ชื่อผู้ใช้, รหัสผ่าน แล้วกด **เปิดในแอป**
- **กระบวนการในแอป:**
  1. เปิด `BrowserWindow` โหลด URL ที่กรอก
  2. หลังโหลดเสร็จ (`did-finish-load`) รันสคริปต์ที่ **inject** เข้าไปในหน้า
  3. สคริปต์จะ:
     - หา `form` และช่อง username / password (จาก id, name หรือ label)
     - หาปุ่มส่ง (จาก id, name, value หรือข้อความปุ่ม)
     - ถ้ามีฟิลด์ฐานข้อมูล (เช่น `<select>`) และผู้ใช้กรอกค่าไว้ ก็เลือกตามนั้น
     - เติม username และ password (ใช้ setter + dispatch event ให้ React/Vue รับค่าได้)
     - หลังจากหน่วงเล็กน้อย: ส่งฟอร์มด้วย `form.submit()` หรือ **คลิกปุ่มส่ง** (ถ้าเว็บใช้ JavaScript  handle การส่ง)

การปรับปรุงที่ทำแล้วเพื่อให้รองรับทั้ง oa.nkbkcoop.com และ led-ck.com:
- รองรับการหาปุ่มส่งจากข้อความ **"เข้าสู่ระบบ"** (นอกจาก "เข้าใช้งาน") เพื่อให้ตรงกับ led-ck.com
- ถ้าหาปุ่มจากข้อความได้ จะ **คลิกปุ่ม** แทน `form.submit()` เพื่อให้เว็บที่ใช้ JavaScript ในการส่งฟอร์ม (เช่น led-ck.com) ทำงานถูกต้อง

## 4. สรุปสั้นๆ

| คำถาม | คำตอบ |
|--------|--------|
| **ระบบตรวจสอบจากอะไร?** | จาก **username** และ **password** (และอาจมีฐานข้อมูล/ตัวเลือกอื่น) ที่ผู้ใช้กรอก |
| **ตรวจสอบที่ไหน?** | ที่ **เซิร์ฟเวอร์** ของเว็บนั้น ผ่าน request (มัก POST) ไปยัง URL ของฟอร์มหรือ API ล็อกอิน |
| **ตรวจสอบยังไง?** | เซิร์ฟเวอร์เทียบกับ **ฐานข้อมูล/ระบบสมาชิก** แล้วออก **session/cookie** ถ้าถูกต้อง |
| **นำมาใช้ในโปรแกรมยังไง?** | ใช้สคริปต์ inject ในหน้าเว็บ: **หา form, ช่อง user/pass, ปุ่มส่ง** จาก id/name/label แล้ว **เติมค่าและส่งฟอร์ม** (หรือคลิกปุ่ม) ให้ตรงกับพฤติกรรมของแต่ละเว็บ |

ไฟล์ที่เกี่ยวข้องในโปรแกรม:
- `electron-main.js`: handler `open-webview-login` และสคริปต์ inject
- `public/app.js`: ปุ่ม "เปิดในแอป" และการส่ง url, database, username, password
- `public/index.html`: ฟอร์มกรอก URL, ฐานข้อมูล, ชื่อผู้ใช้, รหัสผ่าน
